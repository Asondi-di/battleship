# Battleship LAN — инструкция (новый формат с отдельным окном игры)

## Что изменилось

- Игра больше **не работает внутри маленького popup-модального окна** расширения.
- Popup теперь работает как **лаунчер**: одна кнопка «Открыть игру».
- Игровой интерфейс открывается в **отдельном окне браузера** и остается активным, пока вы сами его не закроете.
- Если окно уже открыто, повторное нажатие кнопки в popup просто фокусирует существующее окно.

---

## 1) Подготовка сервера

1. Откройте терминал в корне проекта.
2. Установите зависимости:

```bash
npm install
```

3. Запустите сервер:

```bash
npm start
```

4. Убедитесь, что порт `3000` доступен в вашей сети.

---

## 2) Установка расширения

1. Откройте `chrome://extensions`.
2. Включите **Режим разработчика**.
3. Нажмите **Загрузить распакованное расширение**.
4. Выберите папку `extension`.

---

## 3) Запуск игры в отдельном окне

1. Нажмите иконку расширения в Chrome.
2. В popup нажмите **«Открыть игру»**.
3. Откроется отдельное окно Battleship LAN.
4. Если окно уже открыто, расширение просто переключит фокус на него.

---

## 4) Подключение к комнате

В открытом игровом окне:

1. Заполните **Сервер WebSocket** (например `ws://192.168.1.20:3000`).
   - Если сервер запущен на **этом же ПК**, можно нажать кнопку **«Авто IP хоста»** — адрес подставится автоматически.
2. Введите ник.
3. Введите код комнаты.
4. Выберите лимит игроков (для хоста).
5. Нажмите:
   - **Создать комнату** — если вы хост;
   - **Подключиться** — если входите в существующую комнату.

---

## 5) Старт матча

1. Нажмите **Авторасстановка**.
2. Нажмите **Подтвердить флот**.
3. Хост нажимает **Запустить игру**, когда все готовы.

---

## 6) Игровой процесс

- Стрелять можно только в свой ход.
- Поле «Атака» показывает выбранного противника.
- Попадания/промахи отображаются цветом на доске.
- Статус вверху подсказывает, чей сейчас ход и когда игра завершена.

---

## 7) Важное поведение окна

- Переключение между вкладками браузера больше **не закрывает игру**.
- Закрытие popup больше **не разрывает сессию**.
- Сессия в комнате сохраняется, пока открыто отдельное игровое окно и активно подключение к серверу.

---

## 8) Если что-то не работает

- Проверьте URL сервера (должен начинаться с `ws://` или `wss://`).
- Проверьте, что сервер запущен (`npm start`).
- Проверьте, что все игроки используют одинаковый код комнаты.
- Проверьте, что порт `3000` не блокируется фаерволом.
